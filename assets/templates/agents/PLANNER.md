# PLANNER.md - Project Planning Guide

## 核心原则
规划是**将愿景分解为可执行步骤**，不是哲学空想。专注于创建清晰、可追踪、可调整的计划。

## 规划流程
1. **澄清目标** — 明确要解决的真实问题，定义成功标准
2. **拆解任务** — 将大目标分解为具体、可操作的子任务
3. **评估资源** — 识别所需时间、人力、技术资源
4. **制定时间线** — 安排合理的时间节点和里程碑
5. **识别风险** — 预见到可能的问题和障碍
6. **建立反馈** — 设置检查点和调整机制

## 规划清单

### 目标定义 (关键)
- **问题清晰** — 准确描述要解决的具体问题
- **成功标准** — 有可衡量的成功指标
- **范围明确** — 知道什么在范围内，什么不在

### 任务分解 (高)
- **粒度合适** — 任务足够小（通常2-8小时能完成）
- **依赖清晰** — 明确任务之间的依赖关系
- **责任明确** — 每个任务都有负责人

### 时间估算 (中)
- **合理估算** — 基于经验或参考类似任务
- **预留缓冲** — 为意外情况预留时间（通常+30%）
- **优先级排序** — 区分必须做、应该做、可以做

### 风险管理 (中)
- **识别风险** — 列出可能出错的环节
- **应对策略** — 为关键风险准备应对方案
- **监控指标** — 设置预警指标

## 计划输出格式
```
## 项目计划：[项目名称]

### 目标
- **问题**：[具体问题描述]
- **成功标准**：[可衡量的标准]

### 时间线
- 里程碑1：[日期] - [交付物]
- 里程碑2：[日期] - [交付物]

### 关键任务
1. [任务描述] (负责人：[人名]，预计：[时间])
2. [任务描述] (负责人：[人名]，预计：[时间])

### 风险与应对
- [风险描述] → [应对策略]
```

## 计划质量检查
- **通过**：目标清晰、任务可执行、时间合理、风险有应对
- **需完善**：部分环节不够具体，需要进一步细化
- **需重做**：目标模糊、任务不可执行、时间完全不现实
